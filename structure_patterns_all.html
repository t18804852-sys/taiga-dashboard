13. データ構造（情報科学）

コンピュータが情報を組織化する基本パターン

スタック（LIFO）

Stack — Last In First Out
最後に入れたものが最初に出る
Undo操作、ブラウザの戻るボタン、再帰呼び出し

キュー（FIFO）

Queue — First In First Out
最初に入れたものが最初に出る
印刷キュー、レジの行列、メッセージキュー

連結リスト

Linked List
各ノードが次のノードを指す鎖
ブロックチェーン、プレイリスト、電車の連結

ハッシュテーブル

Hash Table / Dictionary
キー→ハッシュ関数→インデックスで即座にアクセス
辞書、キャッシュ、データベースインデックス

ヒープ（優先度キュー）

Heap / Priority Queue
親が常に子以上（以下）。最大/最小を即取得
タスクスケジューラ、救急トリアージ

グラフ（隣接行列/リスト）

Adjacency Matrix / List
グラフの内部表現：行列 or リスト
SNS関係、路線図、依存関係

14. 代数・順序構造

数学が発見した「演算と関係」の構造

群（グループ）

Group
集合+演算+4つの公理。対称性の言語
回転群、置換群、暗号（楕円曲線群）

半順序（ポセット）

Partial Order / Poset
すべての要素対が比較可能とは限らない順序
集合の包含関係、タスクの依存関係

束（ラティス）※数学の

Lattice (Algebraic)

上限と下限を持つ
任意の2要素が合併(∨)と交差(∧)を持つ順序構造
論理のAND/OR、集合のunion/intersection

圏（カテゴリー）

Category
「構造の構造」を研究する数学。射の合成
関数の合成、型システム、プログラミングのモナド

15. 関係構造（カーディナリティ）

エンティティ間の「何対何」の関係

1対1（One-to-One）

1:1 Relationship
1つの実体が1つの実体にのみ対応
人↔パスポート、国↔首都

1対多（One-to-Many）

1:N Relationship
1つの実体が複数の実体に対応
著者↔本、部門↔社員、親↔子

多対多（Many-to-Many）

M:N Relationship
複数の実体が複数の実体に対応
学生↔授業、タグ↔記事、俳優↔映画

16. 物語・音楽構造

時間芸術が発見した「感情の構造」

三幕構成

Three-Act Structure
設定→対立→解決。物語の最も基本的な骨格
ほぼすべての映画・小説・プレゼン

起承転結

Kishōtenketsu
日本独自の4段構成。「転」が対立でなく「意外な転換」
漢詩、4コマ漫画、プレゼン

ヒーローズ・ジャーニー

Hero's Journey (Monomyth)
キャンベルの単一神話。日常→非日常→帰還の円環

ソナタ形式

Sonata Form / ABA'
ABA'形式。提示→展開→変容して再現
ベートーヴェン交響曲、ジャズのテーマ→即興→テーマ

Verse-Chorus構造

Verse-Chorus / ABABCB
反復（コーラス）と変化（ヴァース）の交互配置
ポップス、ロック、Jポップ

フーガ（対位法）

Fugue / Counterpoint
同一テーマが時間差・音程差で重層的に展開
バッハのフーガ、カノン、輪唱

17. 言語構造

人間が意味を組み立てる構造

統語木（構文木）

Syntax Tree / Parse Tree
文の階層構造を木で表現
自然言語の文法、プログラムのAST

埋め込み構造（再帰的文）

Recursive Embedding
文の中に文を埋め込む再帰。チョムスキーの核心
関係節、引用、条件文の入れ子

語順類型（SOV / SVO）

Word Order Typology
主語(S)・目的語(O)・動詞(V)の配列パターン
SOV(日本語)、SVO(英語)、VSO(アラビア語)

18. テセレーション（平面充填）

隙間なく平面を埋め尽くすパターン

正則テセレーション（3種のみ）

Regular Tessellation
正多角形だけで平面を埋められるのは3種類だけ

ペンローズ・タイリング（非周期的）

Penrose Tiling (Aperiodic)
隙間なく充填するが周期性がない。準結晶の構造
準結晶、イスラムの幾何学模様

壁紙群（17種）

Wallpaper Groups (17 types)
平面上の繰り返しパターンは数学的に17種類しかない
壁紙、布地、アルハンブラ宮殿

組紐（ブレイド）
